Frases estranhas nos documentos

Inicialmente, é necessario dizer que minha lingua nativa não é inglês, então é possivel que algumas frases pareçam estranhas, pois passaram pela tradução do google tradutor, que por sua vez, não é perfeito.

Pode copiar e usar como quiser o template da documentação

O segundo ponto é que, apesar de essa documentação estar com um bom visual, eu não à fiz para usar como template, ela tem como objetivo unicamente orientar você sobre como utilizar e alterar partes do código do template vendido e onde está cada coisa. Caso deseje usar o template dessa documentação para outra finalidade, sinta-se a vontade, ele é totalmente seu, mas como ele não foi feito para isso, eu não tive o cuidade de organizar os codigos e deixar totalmente legiveis para terceiros, e apesar de eu não ter usado tecnicas de minificação do arquivo nessa documentação, ainda é um pouco ruim de ler o código, então se achar que vale a pena copiar, pode fazer sem problemas.

Bugs do navegador

Podem ocorrer bugs quando a resolução da tela é modificada abruptamente, não estilizando da forma determinada correta, entretanto, esse bug é algo dos navegadores, mas de qualquer forma, o usuario dificilmente muda o tamanho de tela do dispositivo costantemente, então mesmo que o bug ocorra, serão poucos os afetados, para resolver o problema é só reiniciar a pagina após a mudança de resolução e tudo volta ao normal.

Sobre usar FrameWorks

O template não foi feito com bootstrap e nem foi pensado caso alguém queira usar qualquer framework que seja pra modificar o template, ou seja, caso você deseje usar algum framework, é interessante que você incule ele ao template e logo em seguida veja se não ocorreu nenhum conflito entra classes que possam bagunçar o template. Em todo caso, foi evitado ao maximo usar classes do Bootstrap, para que pelo menos ele possa ser usado, entretanto, não asseguro que não usei nenhum classe do bootstrap.

Padões para ler o código

Todo o documento HTML segue boas práticas de identação, então foi atribuido os documentos css antes de qualquer coisa, em seguida o conteudo do template e por fim os documentos de script. Todo o codigo está identado de acordo com boas práticas, seguindo um padrão de arvore da identação.

Foi usado um padrão para nomear as seções, então se você reparar, todas as seções (com excessão da seção "about") serão "Nome da seção" seguidos de "-section"


Se você usa um editor como o "sublime text", você pode se orientar também onde começa e termina as seções observando onde são os pontos em que a identação fica mais a esquerda.

Padões para ler o código

Semelhante ao HTML, nos documentos CSS foi usado também um estilo de arvore de indentação para organizar melhor o codigo, sendo que, o elemento que estiver mais a esquerda são elementos pai, e os que estão mais a direita são elementos filhos, caso estejam no mesmo nivel de identação, ou seja, nem a direita ou esquerda, quer dizer que são elementos do mesmo nivel na arvore de identação.


Os documentos estão divididos pelas suas respectivas seções, facilitando assim encontar elementos mais especificos e a navegação entre os documentos.

Padões para ler o código

Os documentos estão divididos pelas suas respectivas seções e funções, facilitando assim encontar elementos mais especificos e a navegação entre os documentos.


Assim como os outros arquivos, os scripts também seguem o padrão de arvore da identação, e como todas as funções são muito simples, muito provavelmente não é necessaria uma explicação muito a fundo sobre cada função.
